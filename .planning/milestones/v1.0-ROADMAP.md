## Overview

v1.0 delivered a hardened, C-native utility suite for a dwm-based desktop environment targeting Arch Linux and Artix Linux. The milestone included a dual-distro installer, battery and brightness C utilities, shell-to-C migration hygiene, and public-release readiness with a comprehensive README.

## Phases

### Phase 1: Install hardening + platform detection

**Goal**: A fresh user can clone the repo and run `./install.sh` on either a clean Arch or clean Artix system and end up with a working environment — or get a clear, actionable error if their distro is unsupported.
**Depends on**: Nothing (first phase)
**Requirements**: INST-01, INST-02, INST-03, INST-04, INST-05, INST-06, INST-07, INST-08, INST-09, INST-10, INST-11, INST-12, MIG-04, BRIGHT-03

- [x] 01-03: Build+install dwm/st/dmenu/slstatus + utils/, dotfiles with backup, verification summary

### Phase 2: Battery + Brightness utilities

**Goal**: A laptop user gets a dunst notification when battery hits 20% (normal urgency) and again at 5% (critical, persistent), and sees a brightness OSD with a progress bar each time they press the brightness keys.

## Milestone Summary

### Key Accomplishments

1. **Dual-distro installer** — One `install.sh` works on both Arch and Artix with proper distro detection, package variants, and service enablement
2. **Battery notifications** — Working `battery-notify` with tiered urgency (normal at 20%, critical at 5%), state machine, and desktop handling
3. **Brightness control** — New `brightness-notify` utility with OSD, progress bar, and flock serialization to prevent fork-storms
4. **Shell→C parity** — Deprecated `scripts/` directory, fixed CLAUDE.md drift, completed parity audit
5. **Public-ready documentation** — README with install instructions, keybinding reference, troubleshooting, screenshots, and VM verification docs
6. **Repo hygiene** — Removed dwm/p1.rej, fixed .gitignore typo, updated all documentation references

### Key Decisions

- Single `install.sh` with `/etc/os-release` detection rather than separate per-distro scripts
- Two separate utilities for battery and brightness (not one integrated feature)
- Keep `scripts/` as deprecated fallback, delete in next milestone
- `brightnessctl` backend over `xbacklight` or raw sysfs
- Dunst OSD for brightness notifications (not custom Xlib overlay)
- README overhaul over PKGBUILD/CI/CHANGELOG for "public-ready"
- Fail loudly on unsupported distros rather than best-effort installation

### Issues Resolved

- Fixed hardcoded `power-profiles-daemon-openrc` in install.sh (broken on Arch)
- Fixed .gitignore typo `.plannig` → `.planning`
- Removed stale dwm/p1.rej patch-reject file
- Updated CLAUDE.md to reference betterlockscreen (not slock) and powerprofilesctl (not pkexec cpupower)
- Fixed Void Linux references in documentation
- Made battery-notify work on desktops (no battery) — exits silently with status 0
- Implemented flock serialization for brightness to prevent fork-storms on key-repeat

### Technical Debt

- `brightnessctl` not yet in install.sh runtime deps (covered by BRIGHT-03 requirement which completed)
- Some phase directories still contain verbose planning files — could be trimmed
- No AUR PKGBUILD (deferred to future milestone)
- No CI build check (manual install.sh validation)
